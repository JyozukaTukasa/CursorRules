# コンテキスト同期 & トレーサビリティ監査 (Sync & Trace)

// turbo
1. **ディレクトリ構成の同期**
   現在のファイル構成をスキャンし、`docs/architecture.md` の記述と乖離がないかチェックする。
   Run: `tree -L 2 src` (or equivalent)

2. **技術スタックの同期**
   `package.json` を読み込み、`GEMINI.md` の技術スタック一覧を最新化する。

3. **トレーサビリティ監査 (Traceability Check)**
   「要件定義書 (`docs/requirements.md`) に紐付かない、謎のコード」が存在しないかチェックする。
   もしあれば、「これはどの要件の実装ですか？」とユーザーに問いかけ、ドキュメントへの追記を促す。

4. **ADR (決定ログ) の提案**
   直近で大きな設計変更があった場合、「この決定を ADR (`docs/adr/xxx.md`) として記録しますか？」と提案する。
