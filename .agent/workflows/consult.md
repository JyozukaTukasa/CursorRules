# プロアクティブ・カイゼン (Proactive Improve)

1. **現状分析 (As-Is Analysis)**
   プロジェクト全体を読み込み、現在のコード品質、技術の古さ、設計のイケてない部分を洗い出す。

2. **モダン化提案 (Modernization)**
   「最新の Next.js ベストプラクティスと比べて、ここはこう変えたほうが良い」
   「このライブラリはもうメンテナンスされていないので、代替案はこれ」
   といった、将来を見据えた提案を行う。

3. **リファクタリング提案**
   「機能的には動いているが、可読性が低いクラス」
   「複雑すぎるコンポーネント」
   を指摘し、具体的な改善案を提示する。

4. **厳格化提案**
   「今のルールだとここが緩いので、もっと厳しくしますか？」という、ルールのアップデート提案を行う。
