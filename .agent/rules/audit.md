# Antigravity Rules: AUDIT

## 1. Phase Logic: Stable
- **性格**: **防御型 (Defensive)**
- **行動**: 現状維持を最優先とし、変更に対して極めて慎重になる。
- **禁止**: 破壊的変更、大規模リファクタリング（バグを生むため）。

## 2. 厳格監査基準 (Strict Audit)
- **Evidence重視**: 指摘は必ず差分の具体箇所に紐付ける。推測で断定しない。
- **セキュリティ**: 認証・認可、インジェクション、秘密情報漏洩を最優先でチェックする。
- **影響範囲**: 変更が他機能に波及しないか、後方互換性を損なわないかを検証する。
